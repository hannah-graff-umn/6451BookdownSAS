Software Lesson 10 Poisson Regression in SAS

Goals

The goal of this software lesson is to learn how to

  • fit a Poisson regression model, with or without an offset term, and obtain estimates for the effect of a predictor in terms of log count/rate and in terms of rate ratios, and
  • obtain model diagnostics to check assumptions and evaluate the model fit (e.g., overdispersion).

10.1 Datasets

10.1.1 Dataset #1: Lung Health Study

Let’s use the Lung Health Study (LHS) again.

Recall that the dataset and data dictionary can be found on Canvas, lhs.csv and lhsdoc.docx, respectively.

proc import
  DATAFILE = "/<filepath>/lhs.csv"
  OUT = lhs
  DBMS = csv
  REPLACE;
  GETNAMES = yes;
run;

You will use this dataset to answer the following question:

  • Model 1: Is the rate of cigarettes smoked per day different after 3 years of a smoking intervention (with and without Atrovent) versus those who have usual care?

The variables of interest are:

  • cigsYr3: the number of cigarettes smoked per day at year 3 of the study (we will create this variable from the AV3CIGS variable), and

  • alphagroup: the intervention of either UC for usual care, SI-A for intervention plus Atrovent, and SI-P for intervention plus placebo.

10.1.2 Dataset #2: Canadian Smoking and Death Study

Another dataset used in this software lesson is the Canadian Smoking and Death Study (CSADS).

The CSADS was a survey administered to male veterans of either World War I, World War II, or the Korean War. The initial survey was administered from September 1955 through June 1956. All men (or their families) were then re-surveyed in June 1962 to determine their survival status.

The dataset and report/article of the study can be found on Canvas, titled CanadaSmoking.csv and CanadianSmokingAndDeathStudy.pdf, respectively.

proc import
  DATAFILE = "/<filepath>/CanadaSmoking.csv"
  OUT = CSADS
  DBMS = csv
  REPLACE;
  GETNAMES = yes;
run;

You will use this dataset to answer the following questions:

  • Model 2: Does the death rate differ for those who smoke cigarettes only verses pipe/cigar only verses smoking both verses not smoking?

  • Model 3: Does the death rate differ for those who smoke cigarettes only verses pipe/cigar only verses smoking both verses not smoking, controlling for age?

Notice that there are 4 variables and what appears to be 36 participants (i.e. 36 rows). However, each row is a summary of participants by age and smoking status. The variable pop is the number of veterans in each age/smoking status group, so it would be better to sum up the values in the pop variable to determine how many veterans were in the study.

proc means data = CSADS sum;
VAR pop;
run;

The output tells us that there were 56,122 veterans in the study.

The variables of interest are:

  • dead: the number who died in the 6 year period from the group,

  • pop: the number in the group,

  • smoke: the smoking status of the individuals in the group. There are 4 categories/levels:

    • no: for those who did not smoke,
    • cigarPipeOnly: for those who smoked either cigars or a pipe but not cigarettes,
    • cigaretteOnly: for those who smoked cigarettes but neither cigars nor a pipe, and
    • cigarettePlus: for those who smoked both cigarettes as well as at least one other form of tobacco, and

  • age: the age group of the veteran (in 5 year increments starting with 40) (Note: will be used in Model 3 only).

10.2 Data

10.2.1 Data cleaning

10.2.1.1 EDIT 1: Creating the cigsYr3 variable in lhs dataset

The AV3CIGS variable measures the number of cigarettes smoked per day. For most participants, this value was reported as a whole number, but approximately 50 participants reported \(\frac{1}{2}\) a cigarette per day. We are going to round those participants up to 1 cigarette per day.

We will use the if-then-else functions to substitute 1 for 0.5. Because we want to preserve the original data, we will create a new variable called cigsYr3.

*data editing and cleaning;
data lhs1;
  SET lhs;
  IF AV3CIGS = 0.5 THEN cigsYr3 = 1;
  ELSE cigsYr3 = AV3CIGS;
run;

*testing code;
proc freq data = lhs1;
 TABLE cigsYr3;
 TABLE AV3CIGS;
run;

The table is very large, but we don’t need to check all of it to make sure the cigsYr3 variable was coded correctly. Instead, we will want to focus on the first three rows and columns. The first three columns total the number of people in the original variable AV3CIGS that smoked 0, 0.5, and 1 cigarettes, respectively. The first three rows are the lowest three values for our new cigsYr3 variable. We see that there is no longer a 0.5 value for this new variable, and instead, all the participants who smoked 0.5 cigarettes have been rounded up to 1 cigarette per day. This is what we wanted, so the recoding worked.

10.2.1.2 EDIT 2: Log-transforming pop variable (offset variable) in CSADS dataset

In this dataset, we have the number of veterans who died in a given age group and smoking category, as opposed to the number of an event that a participant engaged in. Therefore, this outcome is a binary variable (died, didn’t die), and the dataset presents the aggregate information. Since the groups are comprised of different numbers of veterans, we can’t just compare the counts. Instead, we need the mortality rate, which is the number of deaths divided by the number of veterans in that group. Since the number of veterans is a separate variable (pop), we will need to use this variable as our offset term.

Recall that Poisson regression models the LOG of the rate. Because the offset variable needs to be on the same scale as the model, we need to log-transform it:

* create log-transformed variable for offset;
data CSADS1;
  set CSADS;
  log_pop = log(pop);
run;

10.3 Model Fitting

To fit a Poisson regression model, use the proc genmod task, specifying the link=log and dist=poisson options in the model statement. There are several other options that are useful in the model statement:

  • type3 requests type 3 statistics for categorical variables.

  • waldci generates confidence limits for the estimated regression coefficients.

10.3.1 Model 1: Count data with a fixed denominator

Let’s see if the treatment groups (SI-A and SI-P) had any affect on the smoking rate (i.e., cigarettes per day) 3 years after the start of the study. The reference group will be the control group (usual care, UC).

Note that since the variable cigsYr3 is the number of cigarettes PER DAY, the data is already in rate form and we do not need an offset term.

proc genmod data = lhs1; 
CLASS alphagroup(ref="UC") / param = ref;
MODEL cigsYr3 = alphagroup / link = log dist = poisson type3 waldci;
run;

The model output is similar to the output for linear regression and logistic regression.

10.3.1.1 Estimate effect of predictors

Recall that in a Poisson regression model, the regression coefficients represent changes in the log count/rate scale. Also recall that you can use the results to estimate rate ratios.

10.3.1.2 Rate Ratio

To estimate the effect of predictors in terms of the rate ratio, take the anti-log of the regression coefficients (i.e., exponentiate the beta coefficients). You can only do this using the estimate statement in proc genmod, specifying the name of the row (e.g., 'Beta XXX'), the predictor of interest, 1 (for numerical variables) or contrasts (for categorical ones), and the exp option.

For the contrasts, you list the values in the order of the categories found in the model. For binary categorical variables, you use 1 and -1, where the category listed as 1 is the “level of interest” and the category listed as -1 is the “reference level”.

For example, let’s look at a binary variable. The AV3GUM has two levels: 0 is not using nicotine gum and 1 using nicotine gum. If you want to compare 1 to 0, you would list the contrasts as -1 1 in the estimate statement.

For multi-categorical variables, you use 1 and -1 for the two levels you want to compare and 0 for the level(s) that isn’t taken into account. For example, the alphagroup variable has three categories: SI-A, SI-P, and UC (in that order in the model output).

  • if you want to compare SI-A to UC (but leave SI-P out), you list the contrasts as 1 0 -1 in the estimate statement.
  • if you want to compare SI-P to UC (but leave SI-A out), you list the contrasts as 0 1 -1 in the estimate statement.
  • if you want to compare SI-A to SI-P (but leave UC out), you list the contrasts as 1 -1 0 in the estimate statement.

The code below shows for each of these estimates, in the order that was just presented:

proc genmod data = lhs1; 
class alphagroup(ref="UC") / param = glm;
model cigsYr3 = alphagroup / link = log dist = poisson type3 waldci;
estimate 'Beta SIA vs UC' alphagroup 1 0 -1 / exp;
estimate 'Beta SIP vs UC' alphagroup 0 1 -1 / exp;
estimate 'Beta SIA vs SIP' alphagroup 1 -1 0 / exp;
run;

There is a new table at the end of the output, titled Contrast Estimate Results, in addition to the usual tables you saw when fitting the Poisson regression model. Each estimate statement will present two rows of information: one for the regression coefficient (Beta XXX) and one for the exponentiated coefficient (Exp(Beta XXX)). The two columns of interest are the L'Beta Estimate, which presents an estimate of the coefficients (log counts or rate ratio, depending on the row) and L'Beta Confidence Limits, which presents confidence limits for the estimates.

10.3.1.3 Rate ratio for multi-categorical predictors

For the categorical variable in the model, you may be interested in analyzing whether there are any differences in the response variable between each group after adjusting for the other predictors in the model. To estimate adjusted effects of a (multi-) categorical variable, use the lsmeans statement and the diff option for multiple comparisons. Options that may be useful include:

  • exp to exponentiate the coefficients.
  • cl to get confidence limits.
  • diff for comparing the levels.
  • adjust=tukey to adjust p-values for multiple comparisons (using Tukey method).

You may be asking why we use the lsmeans statement if one can estimate the rate ratio from the model output. The model output only compares each level of a categorical variable to the reference level. For example, it doesn’t give us the rate ratio comparing the SI-P level to the SI-A level. Recall that the lsmeans statement can provide pairwise comparisons for all levels.

proc genmod data = lhs1; 
class alphagroup / param = glm;
model cigsYr3 = alphagroup / link = log dist = poisson type3 waldci;
lsmeans alphagroup / exp cl diff adjust = tukey;
run;

The table of interest for comparing all categories of a variable is the last table, labeled Differences of [variable] Least Squares Means Adjustment for Multiple Comparisons: Tukey-Kramer. The values presented in that table include the difference in the log rate between the two levels in that row (Estimate), the model-estimated standard errors for the estimate (Standard Error), the z-statistic (z Value), and its p-value, before (Pr > |z|) and after Tukey’s adjustment for multiple comparisons (Adj P). It also presents the rate ratio between the two levels (Exponentiated) and it’s upper and lower confidence interval, before (Exponentiated Lower and Exponentiated Upper) and after Tukey’s adjustment for multiple comparisons (Adj Lower Exp and Adj Upper Exp).

10.3.2 Model Evaluation

Poisson models assume that the response variable come from a Poisson distribution, which, by definition, assume that the variance (i.e., \(sd^2\)) is equal to the mean. Because of this, we need to check for overdispersion in our model.

Overdispersion happens when the observed variance is larger than the expected (or assumed) variance. To check for overdispersion, you evaluate the ratio of the residual deviance to its degrees of freedom. When this ratio is significantly greater than 1, then its an indication of overdispersion.

Looking at the table called Criteria For Assessing Goodness of Fit in the proc genmod output, we can see that the Deviance was 91516 and its degrees of freedom was 5463. Therefore, the ratio of these values equals 16.75. Is this value significantly larger than 1? You can test this by using the Chi-square distribution to find the p-value for the given Pearson Chi-Square value and the associated degrees of freedom.

data pvalue;
  df = 5463; 
  chisq = 91516;
  pvalue = 1 - probchi(chisq, df);
run;

proc print data = pvalue;
run;

10.3.3 Models for overdispersed data

So the model shows evidence of overdispersion. Now what? There are two ways to handle overdispersion.

10.3.3.1 Quasi-Poisson regression

One option is to use a quasi-Poisson regression model, which can be added to the proc genmod task by specifying the scale=pearson option in the model statement.

proc genmod data = lhs1; 
class alphagroup(ref="UC") / param = ref;
model cigsYr3 = alphagroup / link = log dist = poisson scale = pearson type3 waldci;
run;

The output is the same as in the regular Poisson regression model. You will notice that while the estimates are the same between the Poisson and quasi-Poisson models, the standard errors are different. As a result, the tests may differ.

10.3.3.2 Negative binomial regression

The second option is to fit a negative binomial regression model with the dist=negbin option in the model statement.

proc genmod data = lhs1; 
class alphagroup(ref="UC") / param = ref;
model cigsYr3 = alphagroup / dist = negbin type3;
run;

10.3.4 Model 2: Binary data with a variable denominator (offset term)

Recall that for fixed denominators (such as what we saw using the number of cigarettes smoked per day at the third annual visit in the LHS dataset), we don’t need to specify an offset term. However, if a variable has a denominator that varies, you will need to to use a Poisson regression with an offset term. So now we are going to shift to a dataset where the outcome is binary (death or not) and an offset term is needed (due to a denominators varying).

For this analysis, we will be predicting the mortality rate in a 6-year period for male veterans in Canada based on the type of smoker they were. Since the data is given as number of deaths, instead of the mortality rate, we will be using the number of participants in the group as our offset term. Since the predictor of interest is the smoking type, we will also calculate the rate ratios for the smoke variable.

proc genmod data = CSADS1; 
class smoke(ref="no") / param = glm;
model dead = smoke / link = log dist = poisson offset = log_pop type3 waldci;
estimate 'cigar/pipe vs no' smoke 1 0 0 -1 / exp;
estimate 'cigarette only vs no' smoke 0 1 0 -1 / exp;
estimate 'cigarette + other vs no' smoke 0 0 1 -1 / exp;
run;

The output is similar to those we’ve seen before.

10.3.5 Model 3: Binary data with a variable denominator and a covariate

It seems counterintuitive that those who smoked cigarettes and other forms of tobacco would not have a significantly different mortality rate than those who don’t smoke, when those who smoke only those other forms of tobacco do. Could it be that many of older people who smoked multiple forms of tobacco didn’t survive to make it into the study? After all, the mortality rates are higher for the older age groups. Let’s add age as a covariate to control for the potential age effect.

proc genmod data = CSADS1; 
class smoke(ref="no") age(ref="40-44")/ param = glm;
model dead = smoke age / link = log dist = poisson offset = log_pop type3 waldci;
estimate 'cigar/pipe vs no' smoke 1 0 0 -1 / exp;
estimate 'cigarette only vs no' smoke 0 1 0 -1 / exp;
estimate 'cigarette + other vs no' smoke 0 0 1 -1 / exp;
run;

The output is similar to those we’ve seen before.
